# CLAUDE.md

This file provides guidance to Claude Code when working in this repository.

## Project Overview

Documentation site for the **Adobe Express Embed SDK** (`cc-everywhere`). Content is markdown-based and served via a local Express dev server for local development. The published site lives on [developer.adobe.com](https://developer.adobe.com/express/embed-sdk/docs/).

## Repository Structure

```
src/pages/          # All documentation content (markdown)
  guides/           # How-to guides and tutorials
  reference/        # LegacyAPI reference docs
  v3/, v4/  # Version-specific docs (v1 and v2 are EOL, v3 is deprecated, v4 is the current version)
  support/          # Support resources
  config.md         # Site navigation configuration
scripts/            # Build and utility scripts
sample/             # Code samples
public/             # Static assets
redirects.json      # URL redirect mappings
api-refs-sidebar.json  # API reference sidebar navigation (generated by devex scripts)
```

## Common Commands

```bash
# Start local dev server (port 3003)
npm run dev

# Lint all content (checks links, markdown style, etc.)
npm run lint

# Lint — errors only (no verbose output)
npm run lint:errorOnly

# Lint — skip dead link checks (faster)
npm run lint:skipdeadlinks

# Lint — dead links only
npm run lint:deadlinksonly

# Build API reference (sync docs → sanitize → generate sidebar)
npm run devex:build-api-reference

# Individual API reference steps
npm run devex:sync-docs          # Import docs from SDK source
npm run devex:eds-sanitize       # Rename files + normalize links
npm run devex:generate-sidebar   # Update api-refs-sidebar.json

# Check redirects
npm run redirectCheck:stage
npm run redirectCheck:prod
```

## Workflow Notes

- **Branch naming**: use `AdobeLDAP--<description>` for personal branches.
- **PRs target `main`**. Run lint before opening a PR.
- Always run `npm run lint:skipdeadlinks` for a quick sanity check before committing doc changes; use `npm run lint` for full validation before a PR.
- `v1` and `v2` docs are intentionally hidden — do not re-enable them.
- When editing `config.md`, take care with navigation ordering; it drives the site's left-nav structure.
- When updating API reference content, use `npm run devex:build-api-reference` to regenerate from source rather than editing generated files by hand.
